Guide développeur

Instructions essentielles pour connecter vos casques VR à VHR Dashboard Pro (USB, puis WiFi en 1 clic) et exploiter le streaming, la voix PC→casque et les sessions multi-utilisateurs.

Prérequis

  • ADB (Android Platform Tools) - Installés et configurés dans le PATH. Télécharger ici.
  • Casque VR compatible - Meta Quest, Pico, HTC Vive Focus, ou tout casque Android avec ADB.
  • Câble USB - Pour la connexion initiale et le débogage.
  • Mode développeur - À activer depuis l'app mobile du casque (si applicable).

Drivers USB / ADB (tous casques Android)

Si votre casque n’est pas reconnu, installez ou mettez à jour les drivers ci-dessous puis rebranchez le câble :

  1. Platform Tools (ADB) : télécharger, dézipper et ajouter le dossier platform-tools dans votre PATH.
  2. Windows : installez un driver USB valide :
    • Driver USB Google (générique, fonctionne pour la plupart des casques Android).
    • Ou le driver constructeur (Meta Quest / Pico / HTC…) depuis le site du fabricant si l’appareil n’apparait toujours pas.
  3. Utilisez un câble USB data de bonne qualité, changez de port USB si besoin, puis acceptez la demande de débogage USB dans le casque.
  4. Vérifiez : 
    adb devices
    L’appareil doit apparaitre device dans la liste. Sinon, redémarrez le casque et relancez l’étape driver.

Connexion initiale (USB)

  1. Vérifier ADB : 
    adb version
adb devices
  2. Activer le mode développeur sur votre casque VR.
  3. Connecter le casque en USB à votre ordinateur.
  4. Accepter le débogage USB sur l'écran du casque.
  5. Vérifier la connexion : 
    adb devices

    Vous devriez voir votre appareil dans la liste.

WiFi en 1 clic (Dashboard Pro)

Dans le Dashboard Pro, après la première connexion USB, ouvrez l'appareil et cliquez sur "Activer WiFi" : le casque bascule automatiquement en ADB over WiFi sans commande manuelle.

Le bouton gère la séquence adb tcpip 5555, la récupération d’IP et la connexion. Débranchez ensuite le câble si vous le souhaitez.

Basculer en ADB over WiFi (fallback manuel)

Si nécessaire, vous pouvez toujours le faire en ligne de commande :

  1. Activer ADB over WiFi : 
    adb tcpip 5555
  2. Trouver l'adresse IP du casque : 
    adb shell ip addr show wlan0

    Notez l'adresse IP (ex: 192.168.1.100).

  3. Connecter en WiFi : 
    adb connect 192.168.1.100:5555
adb devices
  4. Déconnecter le câble USB (optionnel).

Utiliser VHR DASHBOARD

Après connexion du casque (USB ou WiFi), le Dashboard Pro détecte l'appareil et vous permet :

  • de lancer le streaming d'écran en HD,
  • de diffuser la voix PC → casque en direct pour guider l’utilisateur,
  • d'utiliser les sessions multi-utilisateurs (prochaine mise à jour),
  • de lancer/arrêter des apps distantes,
  • de gérer les profils (qualité/latence) et l'enregistrement local.

Pour tout dysfonctionnement, consultez la page Contact ou signalez le probleme via la page de support.

Depannage

  • "no devices/emulators found" - Verifiez que le casque est en mode developpeur, connecte en USB, et que le debogage USB est active.
  • Connexion WiFi instable - Assurez-vous que le casque et l'ordinateur sont sur le meme reseau. Reconnectez en USB et relancez adb tcpip 5555.
  • ADB non reconnu - Verifiez que les Android Platform Tools sont installes et configures dans le PATH. Redemarrez votre terminal apres installation.
